前言

要使用 Thymeleaf 模板语言，首先需要在 pom.xml 文件里引入其依赖，否则会报 404 错误！

如果要在 IDEA 中自动提示 Thymeleaf 语法，需要在 html 标签中引入其命名空间。

th:* 是 HTML 5 规范不允许的。因此要添加命名空间避免 IDE 提示语法错误。

若要遵守 HTML 5 规范，可以切换到 Thymeleaf 的数据属性语法：使用 data-th-* 来定义属性。

th:* 更通用，并且在每个 Thymeleaf 模板模式（XML，TEXT…）中都允许，而该 data- 仅在 HTML 模式中允许。

常用属性和表达式

th:text & th:utext

消息表达式 #{…}

#{...} 表达式可以获取到某个配置文件中的属性，可以实现 i18n 的功能。

如果配置文件中的信息有占位符，则可以在后面添加一个参数来传递。多个参数逗号隔开。

链接表达式 @{…} 与 th:href

使用场景如下：

绝对 URL：http://www.thymeleaf.org

相对 URL：

  • 页面相关：user/login.html
  • 上下文相关：/itemdetails?id=3
  • 服务器相关：~/billing/processInvoice
  • 协议相关：//code.jquery.com/jquery-2.0.3.min.js

上下文相关 URL：传入上下文链接时，会自动添加服务器中的上下文名称。
服务器相关 URL：允许在同一服务器的另一个上下文（应用程序）中调用 URL。

官网案例：

  • 允许对 URL 参数使用表达式：@{/order/details(orderId=${o.id})}
  • 多个参数用逗号隔开：@{/order/process(execId=${execId},execType='FAST')}
  • 路径中允许使用变量模板：@{/order/{orderId}/details}
  • 以 / 开头的相对 URL 会自动填充上下文名称：/order/details
  • 可以使用 Servlet API 中的机制为每个 URL 插入重写过滤器。

数据转换 & 格式化

预处理

预处理是在正常表达式之前执行的表达式，它可以修改最终的表达式。

若要使用预处理表达式，只需要在表达式前后都加上双下划线即可。

预处理字符串 '__'可以在属性中使用，需要用斜杠转义：'\_\_'。

迭代 th:each

后端传入可迭代数据时，可以使用 th:each 属性来迭代。

下列类型数据可以使用 th:each 进行迭代：

  • 任何实现 java.util.Iterable 的对象。
  • 任何实现 java.util.Enumeration 的对象。
  • 任何实现 java.util.Iterator 的对象。其值将在迭代器返回时使用，无需将所有值缓存在内存中。
  • 任何实现 java.util.Map 的对象。当迭代 Map 时，迭代器变量类型为 java.util.Map.Entry。
  • 任何实现 java.util.stream.Stream 的对象。
  • 任何数组。
  • 任何其他对象都将被视为包含对象本身的单值列表。

迭代状态变量

在使用 th:each 时，Thymeleaf 提供了一种有助于跟踪迭代状态的机制：状态变量。

  • index：当前迭代索引，从 0 开始。
  • count：当前迭代索引，从 1 开始。
  • size：迭代变量中的元素总数。
  • current：每次迭代的 iter 变量。
  • even/odd：当前迭代是偶数还是奇数。
  • first：当前是否是第一个迭代。
  • last：当前是否是最后一个迭代。

条件判断

简单条件语句 if & unless

th:if 条件成立才渲染该 DOM。

th:if 判断 true/false 的几种情况：

  • 布尔值的 true 为 true。
  • 非 0 的数字都是 true。
  • 非 '' 的字符都是 true。
  • 非 'false'、'off'、'no' 的字符串都是 true。
  • 布尔、数字、字符、字符串以外的类型都是 true。
  • 如果值为 null，则判断为 false。

th:unless 条件不成立才渲染该 DOM。

switch & case

和其他语言中的 switch-case 类似，唯一不同的是默认条件是 th:case="*" 而不是 default。

删除片段 th:remove

可以删除一块片段。

th:remove 有 5 种行为方式：

  • all：删除包含标签及其所有子标签。
  • body：不删除包含的标签，但是删除子标签。
  • tag：删除包含的标签，但不删除子标签。
  • all-but-first：删除第一个子项以外的其他所有子项。
  • none：什么也不删除。可以用来做动态判断。
