从 Vue 2 到 Vue 3

发表于2023-02-19|更新于2023-02-19
|字数总计:5.9k|阅读时长:23分钟

前言

之前一直都是写的 Vue 2，现在开始接触 Vue 3，整个人都迷糊了…，感觉语法上变化挺大的。Vue 3 新特性学过一遍，但是没动手写项目，加上两个月没写代码，大概都忘掉了吧。现在重新学一遍 Vue 3，想着把这些东西记录下来，方便以后忘了再次学习~

Vue 3 的变化

性能提升

官方数据为：打包大小减少 41%，初次渲染速度快 55%，更新渲染速度快 133%，内存减少 54% 等。

总结：性能提升、内存减小、速度更快。

源码升级

  1. 使用 Proxy 代替 defineProperty 实现响应式。
  2. 重写虚拟 DOM 的实现和 Tree-Shaking。

TypeScript

Vue 3 可以更好地支持 TypeScript。

新特性

Composition API (组合API)

  1. setup 配置
  2. ref 与 reactive
  3. watch 与 watchEffect
  4. provide 与 inject

新的内置组件

  1. Fragment
  2. Teleport
  3. Suspense

其他改变

  1. 新的生命周期钩子
  2. data 选项应始终被声明为一个函数
  3. 移除 keyCode 支持作为 v-on 的修饰符

Vue 3 是先创建一个 app，再挂载 DOM。

常用 Composition API

setup

setup 是 Vue 3 中的一个新的配置项，其值为一个函数。它是所有 Composition API “表演的舞台”。

组件中所有用到的：数据、方法等等，都要配置在 setup 中。

setup 函数的返回值有两种：
若返回一个对象，则对象中的属性、方法等在模板中可以直接使用；
若返回一个渲染函数，则可以自定义渲染内容。
开发中常用的做法是：将模板中使用到的数据、方法等在 setup 中返回出去。

注意点：
setup 函数执行时机在 beforeCreate 之前。
在 Vue 3 中也可以使用 Vue 2 的标签，但是不推荐混用。
在 Vue 2 的配置中（data、methods等）可以访问 setup 的属性和方法。
在 Vue 3 的 setup 中不能访问 Vue 2 配置中的内容，在 setup 函数中 this 为 undefined。
如果 Vue 2 配置项和 Vue 3 setup 函数里有重名的属性或方法，以 setup 中的为准。
setup 不能是一个 async 函数，模板看不到 promise 中的属性。（但是可以在异步部组件中使用）

关于 setup 函数的参数：

setup 函数可以接收两个参数，第一个是 props，第二个是 context。

在 Vue 2 中，父组件向子组件传参通常是直接将属性写在子组件的标签中，然后在子组件的 props 配置项中接收属性即可使用。若子组件没有在 props 配置项中接收，Vue 2 也会将属性保留起来。每个 VueComponent 实例对象中都有一个 $attrs 属性，这里就保存了没有被 props 配置项接收的属性(接收过的属性不会出现在 $attrs 中)。因此，就算子组件没有使用 props 配置项，也可以在函数中使用 this.$attrs 拿到传过来的属性。但是这种办法并不能对传入属性做出限制，因此几乎不会用这种办法。

slots 与 props 同理。如果父组件向子组件传入了 DOM 节点，且子组件并没有设置 slot 的位置，没有使用这个 slot，那么 Vue 2 就会将传入的 slot 虚拟节点保存在 VueComponent 实例对象的 $slots 属性中。

在 Vue 3 中父组件向子组件传参时，如果子组件配置了 props 项，那么 setup 的第一个参数就是一个保存了所有 props 数据的 Proxy 实例对象。如果没有配置 props 项，Vue 3 会将传入的参数保存在 setup 的第二个参数中(context)。

context 里面有 3 个属性，attrs、emit、slots，分别用来保存未定义的属性、自定义事件、插槽。

ref

在 Vue 2 中也有 ref，其作用是使用在 DOM 节点中，然后在 methods 中就可以直接拿到 DOM 节点了，类似一个 ID 的作用。

在 Vue 3 中仍然可以这么使用，只是在 Vue 3 中多了一个 ref 函数，作用是将基本类型数据变成响应式。

ref 函数处理基本类型数据之后会返回一个 RefImpl 对象，后续使用数据需要对其 value 属性进行操作。
在模板中使用这些数据时，会自动解析其 value 属性，因此在模板中可以直接使用属性名。
ref 处理基本数据类型的响应式依然是靠 Object.defineProperty() 的 get 和 set 完成的。

ref 处理对象类型数据时，会借助 reactive 函数来完成。

toRef & toRefs

toRef 的作用就是将对象里的一个普通属性转换成一个响应式属性。

toRefs 的作用就是将对象里的所有属性转换成响应式属性。

在 setup 函数中返回出去的数据必须要是响应式的，才能在模板中使用。即用 ref 处理普通属性，用 reactive 处理对象属性。
而返回响应式的对象数据在模板中使用时，需要以对象的名称点属性的方式来使用(person.name, person.age)，这样编码比较麻烦。
因此，在 setup 函数返回时，可以使用 toRefs 将对象的所有属性转换成响应式，然后将其解构，就能将对象里的所有属性返回出去了。

reactive

reactive 的作用是将对象类型数据变成响应式。传入基本数据类型控制台会报警告。

将对象传入 reactive 函数之后，会得到一个 Proxy 对象，来实现数据的响应式。

得到的 Proxy 对象可以像普通对象一样使用，支持多层嵌套的对象数据和数组数据。

由于使用 ref 函数来做基础数据的响应式时，访问的时候需要大量用到其 value 属性，增加无意义代码，所以开发中常会将所有使用到的基本数据或对象数据封装到一个对象中，然后使用 reactive 函数对这个对象做响应式，然后将这个对象解构返回，这样所有数据都可以直接正常使用了，不再需要使用 value 属性。

Vue 2 响应式原理

对象类型的响应式是通过 Object.defineProperty() 函数对属性的读取、修改进行拦截来实现的，也叫做数据劫持。
数组类型的响应式是通过 重写更新数组的一系列方法 来实现拦截的，对数组的变更方法进行了包裹。
这样也产生了一些问题，如 对象中新增属性、删除属性时，界面不会更新； 数组通过下标修改时，界面也不会更新。

Vue 3 响应式原理

在了解 Vue 3 响应式原理之前，需要先认识一个对象：Reflect。

Reflect 是一个内置的对象，它提供拦截 JavaScript 操作的方法，是 ES6 为了操作对象而提供的新 API。

现阶段，某些方法同时在 Object 和 Reflect 对象上部署，未来的新方法将只部署在 Reflect 对象上。

修改某些 Object 方法的返回结果，让其变得更规范化。如 Object.defineProperty(obj, name, desc) 在无法定义属性时，会抛出一个错误，而 Reflect.defineProperty(obj, name, desc) 则会返回false。

Reflect 对象的方法与 Proxy 对象的方法一一对应，只要是 Proxy 对象的方法，就能在 Reflect 对象上找到对应的方法。

computed

Vue 3 中的计算属性相较于 Vue 2 的区别不大，只是变更了使用方式，Vue 2 的计算属性是写在 computed 配置项中的，而 Vue 3 是通过 computed 函数来实现的。

watch

在 Vue 3 中，watch 的使用方法和 computed 类似，都是引入使用其对应的函数。

但是在 Vue 3 中，由于属性定义的方式有了变化，watch 的使用细节上也就有一些变化。

watchEffect

watchEffect 也是用来监视属性的，它比 watch 更智能一些。

watch 既要指明监视的属性，也要指明监视的回调。
watchEffect 只需要传入一个回调函数，函数里面用到了哪个属性，就相当于监视了哪个属性。

watchEffect 有点像 computed，但是 computed 更注重计算出来的值，而 watchEffect 更注重过程，没有返回值。

Vue 3 生命周期

Vue 3 的生命周期相较于 Vue 2 没有太大的变化。

Vue 3 在生命周期钩子的使用上有所变化。

Vue 2 中的生命周期钩子：

beforeCreate, created, beforeMount, mounted, beforeUpdate, updated, beforeDestroy, destroyed

Vue 3 中可以继续使用 Vue 2 的生命周期钩子，但是有两个被更名：
beforeDestroy ==> beforeUnmount
destroyed ==> unmounted

Vue 3 中也提供了 Composition API 形式的声明周期钩子：

setup, onBeforeMount, onMounted, onBeforeUpdate, onUpdated, onBeforeUnmount, onUnmounted

如果 Composition API 的生命周期钩子函数与生命周期配置函数一起使用，两个都会执行，且 Composition API 的函数优先执行。

setup 函数在所有声明周期钩子函数之前执行。

自定义 hook

Vue 3 中有个叫自定义 hook 的说法，大致就是将某个功能单独封装在一个 js 文件中，其他组件需要使用时引入这个 js 即可使用，类似 Vue 2 中的 mixin。（虽然不知道为什么要叫 hook）

常用的做法是：在 src 目录下创建一个 hooks 的目录，里面存放各个 hook 文件，一般取名为 useXXX.js。

其他 Composition API

shallowRef & shallowReactive

shallowRef: 只处理基本数据类型的响应式，不处理对象的响应式。

shallowReactive：只处理对象最外层属性的响应式，不处理嵌套属性的响应式。

如果一个对象数据，后续不会修改其属性，而是直接替换整个对象，则可以使用 shallowRef。
如果对象结构比较深，但是确定只有最外一层数据会发生变化，则可以使用 shallowReactive。

readonly & shallowReadonly

readonly & shallowReadonly 两个函数都是让一个响应式数据变为只读。

区别在于：shallowReadonly 只会让响应式对象的第一层数据变为只读，深层的数据仍然可以修改。

在拿到别人的数据时可以对数据进行保护。

toRaw & markRaw

toRaw 函数的作用是将一个 reactive 生成的响应式对象转换成普通对象。

markRaw 函数的作用是标记一个对象，让其永远不会再成为响应式对象。

将一个对象添加在一个响应式对象中时，Vue 3 会自动将其转换成响应式。如果这个对象无限大，且并不需要修改，那么这个处理成响应式的时间就是可以优化的，用 markRaw 函数标记即可跳过转换。

customRef

customRef 的作用是创建一个自定义的 ref，并对其依赖项跟踪和更新触发进行显示控制。

provide & inject

在开发中经常遇到这么一种情况：组件 A 里面使用组件 B，组件 B 里面又使用了组件 C，形成组件的嵌套关系。

此时，如果组件 A 要传递数据给组件 C，最笨的办法就是先用 props 将数据传给组件 B，组件 B 再用 props 传给组件 C。

或者将数据挂在全局 Vue 实例中，以及使用第三方插件如 Vuex 等办法。这些办法或多或少都会产生一些冗余的逻辑或代码。

在 Vue 3 中，可以使用 provide & inject 两个函数，来非常简单地解决这个问题。

provide：将数据提供出去。父组件可以使用此函数将子组件需要用到的数据提供出去。

inject：将数据注入进来。嵌套任意层次的子组件都可以通过此函数拿到父组件提供的数据。

响应式数据的判断

isRef：判断数据是否是由 ref 创建的对象。
isReactive：判断数据是否是由 reactive 创建的对象。
isReadonly：判断数据是否是由 readonly 创建的对象。
isProxy：判断数据是否是由 reactive 或 readonly 创建的对象。

新的组件

Fragment

在 Vue 2 中，组件必须要有一个根标签。
在 Vue 3 中，组件可以没有根标签，内部会将多个标签包含在一个 Fragment 虚拟元素中。
好处：减少标签层级，减小内存占用。

Teleport

Teleport 组件可以将其内的 HTML 结构移动到指定位置。

to 参数内可以写 CSS 选择器，来选择接收的节点。
对话框这类 HTML 结构不属于任何组件，可以将其传送至 body 标签内，方便控制样式。

Suspense

Suspense 组件可以在异步组件加载完毕之前，准备一个临时的方案进行展示。

异步组件：使用 defineAsyncComponent 函数加载的组件。

使用异步加载组件时，在组件加载成功之前，其 DOM 结构不会展示。

此时就可以使用 Suspense 组件来展示“此处有内容”的临时方案。

如果一个组件被异步引入，那么在 setup 函数中就可以返回一个 Promise 对象。也可以使用 async、await 修饰。
Suspense 组件是通过插槽来实现的，所以需要使用内定的名字：default、fallback。
